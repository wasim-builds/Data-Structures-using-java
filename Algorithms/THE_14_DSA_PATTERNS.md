# 🧩 The 14 Must-Know DSA Patterns

Instead of blindly solving thousands of LeetCode problems, top MAANG engineers learn to recognize the underlying **pattern** of a question within the first 30 seconds.

If you master these 14 patterns, you can solve almost any coding interview question.

### 1. Sliding Window
- **Usage**: Find the longest/shortest/optimal subarray or substring.
- **Keywords**: "Contiguous subarray", "substring", "max sum of size K".

### 2. Two Pointers
- **Usage**: Searching for a pair of elements in a *sorted* array or string.
- **Keywords**: "Sorted array", "find a pair", "palindrome".

### 3. Fast & Slow Pointers (Floyd's Tortoise & Hare)
- **Usage**: Detecting cycles in a LinkedList or finding the middle element.
- **Keywords**: "Cycle", "linked list", "middle".

### 4. Merge Intervals
- **Usage**: Dealing with overlapping time intervals.
- **Keywords**: "Overlapping", "intervals", "meetings", "scheduling".

### 5. Cyclic Sort
- **Usage**: Finding missing or duplicate numbers in an array containing numbers in a given range (e.g., 1 to N).
- **Keywords**: "Array of size N", "numbers from 1 to N", "missing number".

### 6. In-place Reversal of a LinkedList
- **Usage**: Reversing links between nodes without using extra memory.
- **Keywords**: "Reverse", "LinkedList", "in-place".

### 7. Tree Breadth First Search (BFS)
- **Usage**: Level-by-level traversal of a Tree using a Queue.
- **Keywords**: "Level order", "shortest path in unweighted graph".

### 8. Tree Depth First Search (DFS)
- **Usage**: Exploring all paths to the leaves using recursion (or a Stack).
- **Keywords**: "Path sum", "all paths", "binary tree".

### 9. Two Heaps
- **Usage**: Finding the median of a stream of numbers. Keep the smaller half in a Max-Heap and larger half in a Min-Heap.
- **Keywords**: "Median", "running stream of numbers".

### 10. Subsets
- **Usage**: Generating all combinations or permutations of a set using Backtracking.
- **Keywords**: "Combinations", "permutations", "all subsets".

### 11. Modified Binary Search
- **Usage**: Searching in a sorted but modified array (e.g., rotated).
- **Keywords**: "Sorted array", "rotated", "O(log N) time".

### 12. Top 'K' Elements
- **Usage**: Finding the largest/smallest K elements. Always use a PriorityQueue (Heap).
- **Keywords**: "Top K", "Kth largest", "K most frequent".

### 13. K-Way Merge
- **Usage**: Merging K sorted arrays or lists.
- **Keywords**: "Merge K sorted lists".

### 14. Topological Sort
- **Usage**: Finding a valid linear ordering of elements that have dependencies (Graphs).
- **Keywords**: "Prerequisites", "scheduling tasks", "dependency graph".
